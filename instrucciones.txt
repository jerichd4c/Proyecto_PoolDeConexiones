proyecto 2 programación visual:

hacer el programa que simule múltiple concurrencia con hilos (el programa del proyecto 1) ahora utilizando un pool de conexiones "creado por el usuario", es decir sin usar librerías externas.

requisitos del pool y pool manager:

1. ahora estará una clase para pool y pool manager
2. la clase para pool será singleton, es decir, solo será una instancia del pool, pool manager si será para cada usuario
3. pool manager tendrá los siguientes métodos:
	*crearPool()
	*getConnection(): conex ; muestra cuantas conexiones hay
	*returnConecction(Conex C)
	*add Conection() extiende clientes
4. pool tendrá los siguientes métodos:
	*connect()
	*exectuteQuery(String S): resultSet
	*exectute(String S)
	*disconnect()
5. el pool tiene que tener las siguientes propiedades:
	*numero máximo de conexiones: ej 100
	*numero mínimo de conexiones: ej 20
	*numero de crecimiento de conexiones: ej 5
6. tiene que ser de alta escabilidad
7. poolManager tiene que apuntar a el mismo pool de conexione
8. se usaran dos archivos.properties para asinarle atributos al pool:
	*myQuery.properties
	*config.properties
9. solo se puedo usar la libreria de jdbc, no usar librerias externas
